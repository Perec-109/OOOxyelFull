# Zara — полный проект сайта

Одностраничный премиальный сайт бренда одежды с рабочей корзиной,
выбором размера, избранным и отправкой заявок/заказов в Telegram
через безопасный серверный прокси (Cloudflare Worker).

## Структура

```
ooxyel-full/
├── index.html
├── config.js             ← ЕДИНЫЙ адрес Cloudflare Worker (см. ниже!)
├── style.css / variables.css / animations.css / adaptive.css / cart.css
├── cart.js                система корзины (localStorage, размеры)
├── catalog.js              модалка товара + панель избранного
├── products.js             данные товаров + рендер каталога
├── contact.js               форма контактов
├── loader.js / navbar.js / cursor.js / scroll.js / animations.js / effects.js / gallery.js / main.js
├── main.c / engine.cpp      учебные заготовки серверной логики (не исполняются в браузере)
├── robots.txt / sitemap.xml SEO
├── 404.html
└── worker/
    ├── index.js            Cloudflare Worker — прокси к Telegram Bot API
    └── wrangler.toml
```

## Быстрый старт: как связать сайт с Telegram (самое важное!)

**Сообщения не приходят в Telegram почти всегда по одной из двух причин:**
адрес воркера не настроен, либо неверный chat_id. Разберём по порядку.

### Шаг 1. Задеплоить Worker

```
cd worker
npm install -g wrangler      # если ещё не установлен
wrangler login
wrangler deploy
```

После деплоя в терминале появится адрес вида:
```
https://ooxyel-contact-proxy.твой-юзернейм.workers.dev
```
Скопируй его.

### Шаг 2. Задать секреты (токен бота и chat_id)

```
wrangler secret put TELEGRAM_BOT_TOKEN
wrangler secret put TELEGRAM_CHAT_ID
```
Wrangler спросит значение — вставь и нажми Enter. Для группы chat_id
отрицательный: `-123456789` или `-100123456789` для супергруппы.
Узнать точный chat_id: добавь в группу `@RawDataBot`, он пришлёт JSON
с полем `chat.id`.

**Важно:** бот должен быть добавлен в группу как участник — иначе
писать туда он не может, даже с правильным токеном.

### Шаг 3. Вписать адрес воркера в сайт — ОДНО МЕСТО

Открой `config.js` в корне проекта и замени плейсхолдер:

```js
window.OOX_CONFIG = {
  WORKER_URL: 'https://ooxyel-contact-proxy.твой-юзернейм.workers.dev',
};
```

Это единственное место, где нужно прописать адрес — и форма контактов,
и оформление заказа из корзины берут его отсюда автоматически.

**Пока адрес не заменён**, сайт сам предупредит:
- в консоли браузера (F12 → Console) будет заметное предупреждение;
- при попытке отправить форму появится понятное сообщение
  "не настроен адрес сервера" вместо невнятной ошибки.

### Шаг 4. Проверить, что всё настроено правильно

Открой в браузере:
```
https://твой-воркер.workers.dev/diag
```
Покажет JSON с точным статусом: валиден ли токен, видит ли бот
указанный chat_id, и если нет — конкретную причину и подсказку,
что делать (`chat not found`, `bot не в группе` и т.п.).

Открыв просто `https://твой-воркер.workers.dev/` (без `/diag`) —
увидишь подтверждение, что воркер вообще работает.

## Деплой сайта на GitHub Pages

Залей все файлы, КРОМЕ папки `worker/` (это отдельный серверный код,
не статика), в репозиторий → Settings → Pages → main branch → root.

## Что умеет сайт

**Каталог и товар**
- Клик по карточке товара открывает модалку: крупное фото, описание
  (материал, крой, посадка — своё для каждой вещи), выбор размера.
- Без выбора размера добавить в корзину нельзя — как в настоящем шопе.

**Корзина**
- localStorage — переживает закрытие вкладки.
- Один товар в разных размерах — разные позиции корзины.
- +/− количество, удаление, автоматический пересчёт суммы.

**Избранное**
- Отдельная иконка и панель в навбаре.
- Список сохранённых товаров, переход к товару, удаление из списка.

**Оформление заказа**
- Форма (имя, телефон, адрес) → полный состав корзины с размерами
  и итоговой суммой улетает в Telegram через Worker.

**Форма контактов**
- Обычная заявка (имя, email, сообщение) — тоже через Worker,
  токен бота нигде не виден на фронте.

**SEO**
- `robots.txt`, `sitemap.xml`, Open Graph / Twitter Card теги —
  красивое превью при шаринге ссылки в мессенджерах/соцсетях.
- ⚠️ В `index.html` (canonical, og:*, twitter:*), `robots.txt` и
  `sitemap.xml` стоит плейсхолдер `YOUR-USERNAME.github.io/YOUR-REPO` —
  замени на свой реальный адрес GitHub Pages перед публикацией.

## Как добавить свои фотографии

У каждого товара в `products.js` есть поле `img`:
```js
{ id: 1, name: 'Coat Noir', ..., img: 'assets/products/coat-noir.svg' }
```
1. Сохрани фото в `assets/products/` (любой формат — jpg/png/webp).
2. Поменяй путь в `img:` на имя своего файла.
3. То же для `about-visual`, `gallery-01..06`, `look-01..04` в `assets/` —
   просто замени файлы с теми же именами на настоящие изображения
   (сейчас там текстовые заглушки-`placeholder`, а не реальные фото).

Бесплатные фото без проблем с авторскими правами: Unsplash, Pexels.

## Отладка

**Форма/заказ не отправляются** → F12 → Console, там будет точная
причина (`Contact form error:` / `Checkout error:`).

**Сообщения не доходят в Telegram** → сначала `/diag` (см. Шаг 4 выше),
потом при необходимости `wrangler tail` для логов в реальном времени.

**Сайт выглядит не так после правки CSS/JS** → проверь консоль браузера
на ошибки синтаксиса — один пропущенный символ в CSS/JS может сломать
всё, что идёт после него в файле.
