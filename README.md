Это твой исходный сайт (index.html, все css/js, заготовки main.c / engine.cpp)

## Что изменилось по сравнению с твоими файлами

1. **`contact.js`** — больше не содержит токен бота и chat_id.
   Вместо прямого запроса к Telegram форма стучится на твой Worker.
   Убран сломанный фоллбэк через `new Image()` (он всегда показывал
   "не удалось отправить", даже если сообщение реально доходило).

2. **`index.html`** — в форме (`<form class="contact__form" ...>`)
   добавлен атрибут `data-worker-url` с плейсхолдером. Туда нужно
   вписать реальный адрес твоего воркера (см. ниже).

3. **`worker/`** — новая папка, отдельно от сайта. На GitHub Pages
   её заливать не нужно (это не статика, а серверный код для Cloudflare).

Все остальные файлы — без изменений, ровно те, что ты загрузил.

## Деплой сайта (как раньше)

Залей всё, КРОМЕ папки `worker/`, в репозиторий GitHub Pages
(Settings → Pages → main branch → root).

## Деплой Worker (один раз, 5 минут, бесплатно)

1. Зарегистрируйся / залогинься на https://dash.cloudflare.com
2. Установи Wrangler:
   ```
   npm install -g wrangler
   ```
3. Залогинься:
   ```
   wrangler login
   ```
4. Перейди в папку `worker/` и задеплой:
   ```
   cd worker
   wrangler deploy
   ```
5. Добавь секреты — токен бота и chat_id (те же, что были раньше в коде):
   ```
   wrangler secret put TELEGRAM_BOT_TOKEN
   wrangler secret put TELEGRAM_CHAT_ID
   ```
6. После деплоя получишь адрес вида:
   ```
   https://ooxyel-contact-proxy.твой-юзернейм.workers.dev
   ```

## Последний шаг — подключить адрес воркера

Открой `index.html`, найди форму контактов (`id="contactForm"`)
и замени плейсхолдер на свой реальный адрес:

```html
<form class="contact__form" id="contactForm" data-contact-form
      data-worker-url="https://ooxyel-contact-proxy.твой-юзернейм.workers.dev"
      style="margin-top:40px;">
```

Готово — токен бота больше нигде не виден на фронте, и форма
отправляет заявки через защищённый прокси.

## Новое: карточка товара, размеры, избранное (catalog.js)

- Клик по любой карточке товара (в "Новой коллекции" и в общем каталоге)
  открывает модалку с крупным фото, описанием и выбором размера.
- Каждый товар получил реальное описание (материал, крой, посадка)
  и свой набор размеров — без выбора размера добавить в корзину нельзя,
  как в настоящем интернет-магазине одежды.
- Корзина теперь различает один и тот же товар в разных размерах как
  отдельные позиции — можно взять одну куртку в M и другую в L.
- Иконка "Избранное" в навбаре открывает отдельную панель со списком
  сохранённых товаров, откуда можно перейти к товару или убрать из списка.
- Заказ, улетающий в Telegram, теперь показывает размер каждой позиции.

## Новое: система корзины (cart.js + cart.css)

Раньше кнопка "+" на карточке товара просто на секунду показывала
галочку — никакой реальной корзины не было, заказать что-то с сайта
было невозможно. Теперь:

- **`cart.js`** — полноценная корзина на localStorage: добавление,
  изменение количества, удаление, сумма. Переживает закрытие вкладки.
- **`cart.css`** — стили иконки корзины в навбаре, выезжающей панели
  (drawer), модалки оформления заказа и toast-уведомлений.
- Иконка корзины со счётчиком появилась в навбаре (`index.html`).
- Кнопка "+" на карточке товара (и в `.products__grid`, и в
  `.collection__grid` — раньше там вообще не было обработчика кликов,
  это тоже починено) теперь реально кладёт товар в корзину.
- При оформлении заказа (имя, телефон, адрес) заявка с полным составом
  корзины и суммой улетает тебе в Telegram через тот же `worker/`,
  что уже обрабатывает форму контактов — просто с другим форматом
  сообщения (`type: "order"` вместо обычной заявки).

**Важно:** в `cart.js` есть свой `WORKER_URL_FALLBACK` — если у формы
контактов (`[data-contact-form]`) уже стоит `data-worker-url`, корзина
автоматически возьмёт его оттуда. Если формы на странице нет — поправь
адрес воркера прямо в начале `cart.js`.

После деплоя обновлённого `worker/index.js` (`wrangler deploy`) заказы
из корзины начнут приходить отдельным, заметным сообщением — со списком
товаров, количеством и итоговой суммой.



Открой в браузере (после деплоя воркера):
```
https://ooxyel-contact-proxy.твой-юзернейм.workers.dev/diag
```
Эта страница без отправки реальных сообщений покажет:
- задан ли токен/chat_id вообще,
- валиден ли токен бота,
- видит ли бот указанный chat_id, и если нет — почему (с конкретной подсказкой).

### Частые причины, почему сообщение не приходит в группу

1. **Неверный формат chat_id.** Для группы chat_id отрицательный:
   `-123456789` (обычная группа) или `-100123456789` (супергруппа).
   Узнать точный chat_id: добавь в группу `@RawDataBot` или `@getidsbot` —
   он пришлёт JSON с полем `chat.id`.
2. **Бот не добавлен в группу.** Без этого писать в неё он не может физически.
3. **Бот не имеет прав писать** (если в группе ограничения для новых участников).

После того как поправишь секреты:
```
wrangler secret put TELEGRAM_CHAT_ID
```
и снова открой `/diag` — отчёт обновится сразу.

## SEO: robots.txt, sitemap.xml и Open Graph

Добавлены три вещи, важные для поисковиков и красивого превью при шаринге:

- **`robots.txt`** — говорит поисковым роботам, что можно индексировать
  весь сайт, кроме служебной `404.html`, и где искать sitemap.
- **`sitemap.xml`** — список разделов сайта с датой обновления, ускоряет
  первую индексацию (сайт одностраничный, поэтому основная ценность —
  сама главная страница; якоря-разделы добавлены как дополнительный сигнал).
- **Open Graph / Twitter Card** теги в `<head>` — теперь при отправке
  ссылки на сайт в Telegram/WhatsApp/соцсети будет красивое превью
  с заголовком, описанием и картинкой, а не голая ссылка.

**⚠️ Обязательно замени перед публикацией** — в файлах `index.html`
(canonical, og:*, twitter:*), `robots.txt` и `sitemap.xml` стоит плейсхолдер
`YOUR-USERNAME.github.io/YOUR-REPO` — замени на свой настоящий адрес
GitHub Pages везде (проще всего — Find & Replace по всем файлам).

## Отладка формы на фронте

Открой консоль браузера (F12 → Console) при отправке формы.
Если что-то не так — там будет `console.error('Contact form error:', ...)`
с конкретной причиной (неверный адрес воркера, неверный токен,
проблема с сетью).
